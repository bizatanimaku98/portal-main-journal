# portal-main-journal

这是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名、网站或项目，仅作为静态 HTML 文件的集中存放与版本管理空间。

## 项目简介

本仓库的主要目的是以结构化方式保存并展示一系列独立的 HTML 页面。这些页面可能包含文档、工具、演示、笔记或其他形式的网页内容。通过 Git 进行版本控制，便于追踪页面变更、回溯历史版本，并可通过 GitHub Pages 等服务直接访问归档内容。

## 目录说明

仓库根目录下通常包含以下内容：

- `index.html`：入口页面，通常用于列出或导航至仓库内其他页面。
- `pages/`：存放主要 HTML 页面的目录，可进一步按主题或用途组织子目录。
- `assets/`：存放页面所需的静态资源，如 CSS 样式文件、JavaScript 脚本、图片等。
- `docs/`：可选目录，用于存放与页面相关的说明文档或附加资料。
- `README.md`：本文件，仓库说明。

实际目录结构可能根据归档内容有所调整，请以仓库实际文件为准。

## 页面归档说明

每个 HTML 页面应独立完整，尽量减少对外部资源的依赖。若页面需要引用本地资源（如样式、脚本），建议将资源文件放入 `assets/` 目录并在页面中通过相对路径引用。

提交新页面时，请遵循以下基本规范：

- 页面文件使用 `.html` 扩展名，命名清晰、简明。
- 页面头部包含适当的 `<!DOCTYPE html>` 声明和 `charset` 设置。
- 避免在页面中嵌入敏感信息或外部追踪代码。
- 如需更新已有页面，请提交新的 commit 并注明变更内容。
